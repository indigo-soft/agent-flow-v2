# Architecture Issue Records (AIR)

AIR — документи, які фіксують **конфлікти між ADR**: коли нове рішення суперечить
наявному, або два існуючих ADR конфліктують при одночасному застосуванні.

**На відміну від ADR**, AIR — завжди тимчасові. Мета AIR — задокументувати конфлікт,
проаналізувати шляхи розв'язання і прийняти рішення. Після того як рішення прийняте
і зафіксоване в зачеплених ADR — AIR вважається вирішеним.

Архів для AIR не потрібен: вирішені AIR залишаються у цій теці з префіксом `done-`
в імені файлу. Так відкриті завжди вгорі списку, вирішені — внизу.

## Коли створювати AIR?

Створіть AIR якщо:

- Новий ADR **прямо суперечить** наявному (наприклад, вимагає Docker, а ADR-017 його забороняє)
- Два існуючих ADR **конфліктують** при одночасному застосуванні
- Зовнішня вимога суперечить поточній архітектурі і потрібно зафіксувати trade-off

**Не створюйте AIR** якщо:

- Між рішеннями просто є залежність (це `Depends on:` в ADR)
- Рішення просто доповнює інше (це `Related to:` або `Extends:`)
- Можна вирішити в коментарі до PR без окремого документа

---

## Lifecycle AIR

```
Open → Resolved
```

| Статус | Означає |
|--------|---------|
| **Open** | Конфлікт виявлено, рішення ще не прийнято — потрібна увага |
| **Resolved** | Рішення прийнято, зафіксовано в зачеплених ADR, файл перейменований у `done-` |
| **Deferred** | Вирішення свідомо відкладено з конкретною умовою повернення |

## Як створити AIR

### 1. Скопіюйте шаблон

```bash
cp docs/adr/templates/AIR-TEMPLATE.md docs/adr/air/air-00N-short-conflict-description.md
```

### 2. Заповніть секції

- **Статус**: `Open`
- **Зачеплені ADR**: таблиця з ADR, що конфліктують
- **Опис конфлікту**: що конкретно суперечить чому і чому це важливо
- **Оцінка серйозності**: Critical / Moderate / Low
- **Альтернативні шляхи розв'язання**: мінімум 2 варіанти з trade-offs
- **Прийняте рішення**: залишається порожнім до вирішення

### 3. Відкрийте PR і обговоріть

AIR — привід для команди явно обговорити конфлікт, а не вирішувати мовчки.

### 4. Після прийняття рішення

1. Заповніть секцію **Прийняте рішення**
2. Оновіть **зачеплені ADR** — додайте у секцію Зв'язки:
   ```markdown
   - Conflict resolved by: [AIR-001](air/done-air-001-....md) — коротко що вирішили
   ```
3. Змініть статус AIR на `Resolved`
4. Перейменуйте: `air-00N-...md` → `done-air-00N-...md`
5. Оновіть таблицю "Вирішені" вище в цьому файлі

---

## Формат назви файлу

```
air-00N-short-conflict-description.md       ← відкритий
done-air-00N-short-conflict-description.md  ← вирішений
```

Вирішені AIR отримують префікс `done-` — завдяки алфавітному сортуванню
відкриті завжди відображаються **вище** вирішених у файловому менеджері та `ls`.
